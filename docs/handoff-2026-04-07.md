## What was fixed

### 1. Imported delta math (`9a7a270`)
**Problem**: Searches table showed `Found +36, Filtered +0, Dupes +0, Imported +7` — the 29 gap was invisible.
**Root cause**: `finalizeSourceRunDeltasForBatch` in `src/jobs/repository.js` was writing `imported_count` as "currently active in queue" (filtering by application status `new/viewed`). The static column already used `importedKeptCount`; the delta used a different metric.
**Fix**: Changed the SQL to `SUM(CASE WHEN COALESCE(e.hard_filtered, 0) = 0 THEN 1 ELSE 0 END)` — jobs in the batch that are not hard-filtered, regardless of review status. Math now holds: `Found = Filtered + Dupes + Imported` per source per run.
**Aggregate total**: The "Enabled Total" Imported row remains the cross-source-deduplicated queue count from `queueMeta.latestRunImportedCount` (wired in previous session).
**Test updated**: `test/run-deltas.test.js` test 17 — expected `importedCount` changed from 0 to 1 with a comment explaining the new semantics.

### 2. Pagination bug (`85d2267`)
**Problem**: Clicking "Next" in the Jobs queue did nothing.
**Root cause**: `useEffect` at line ~1122 in `App.jsx` had `jobsPage` in its dependency array. Clicking Next → `setJobsPage(2)` → effect re-fired → selected job still on page 1 → `setJobsPage(1)` → instant reset.
**Fix**: Removed `jobsPage` from that effect's deps. The effect should only fire when `selectedJobId` or `sortedJobs` changes (to follow a selected job), not when the user manually pages. Used functional `setJobsPage((current) => ...)` as the guard against redundant setState.

### 3. Pre-run auth preflight (`0321948`)
**Problem**: Indeed (and potentially LinkedIn, YC, Wellfound) captured with unauthenticated sessions, returning generic/public results silently.
**Context**: `runAuthPreflightForEnabledSources` + `runSourceAuthProbe` + `buildAuthPreflightFailure` + `recordBlockedAuthPreflightAttempt` all existed but were never called.
**Fix**: Wired into two run paths in `src/review/server.js`:
- `runAllCapturesWithOptions`: after bridge starts, probes all auth-required live sources, builds `blockedSourceIds` set, skips blocked ones in the capture loop with `outcome: "challenge"`.
- `runSourceCaptureWithOptions`: after `ensureBridgeForSources`, probes the single source if `isSourceAuthRequired(source.type)`; throws with `{ payload: { requiresAuthCheck: true } }` on failure.

**Auth-required types** (from `src/onboarding/source-access.js`): `linkedin_capture_file`, `yc_jobs`, `wellfound_search`, `indeed_search`, `remoteok_search`.
ZipRecruiter (`ziprecruiter_search`) does NOT require auth — not checked.
